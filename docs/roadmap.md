# Roadmap Hermes Messenger

## Этап 0 — План

Сделано:

- README;
- архитектура;
- MVP;
- база данных;
- API;
- Hermes-интеграция;
- безопасность.

## Этап 1 — Mock-MVP frontend

Цель: быстро показать рабочий интерфейс.

Функции:

- список чатов;
- личный чат;
- группа;
- канал;
- HermesBot;
- отправка сообщений;
- имитация ответа;
- localStorage;
- мобильный интерфейс.

Без backend.

## Этап 2 — Backend MVP

Цель: сделать реальную историю сообщений.

Функции:

- регистрация;
- логин;
- SQLite;
- REST API;
- список чатов;
- история сообщений;
- создание сообщений.

## Этап 3 — Realtime

Цель: сообщения как в Telegram.

Функции:

- WebSocket;
- мгновенная доставка;
- статус “печатает…”;
- онлайн-статус;
- reconnect.

## Этап 4 — Groups and channels

Цель: полноценные чаты.

Функции:

- группы;
- каналы;
- участники;
- роли;
- права доступа;
- системные сообщения.

## Этап 5 — Bots

Цель: боты внутри мессенджера.

Функции:

- HermesBot;
- команды;
- обработчики;
- статус бота;
- ответы ботов;
- bot API.

## Этап 6 — Hermes integration

Цель: настоящий Hermes внутри приложения.

Функции:

- backend-прокси;
- Hermes API server или gateway;
- лимиты;
- контекст диалога;
- `/reset`;
- обработка ошибок.

## Этап 7 — Android APK

Цель: мобильное приложение.

Функции:

- Capacitor;
- сборка APK;
- адаптация интерфейса;
- push-уведомления позже.

## Этап 8 — Production

Цель: стабильный мессенджер.

Функции:

- PostgreSQL;
- HTTPS;
- авторизация;
- файлы;
- уведомления;
- мониторинг;
- резервные копии;
- безопасность.

## Рекомендуемый порядок

Для быстрой пользы:

1. mock-MVP;
2. backend;
3. WebSocket;
4. HermesBot;
5. APK.
