# Интеграция с Hermes

## Цель

Подключить Hermes Agent как AI-бота внутри мессенджера.

Бот будет называться:

```txt
HermesBot
```

## Правильная схема

```txt
Пользователь → Frontend → Backend → Hermes Gateway / API Server → Hermes Agent
```

## Неправильная схема

```txt
Пользователь → Frontend → Hermes API
```

Так нельзя, потому что токен Hermes окажется в браузере.

## Где хранить ключи Hermes

Ключи и токены хранить только на backend:

```txt
.env
```

Пример переменных:

```txt
HERMES_API_BASE_URL=...
HERMES_API_KEY=...
HERMES_DEFAULT_MODEL=...
```

## Как пользователь обращается к HermesBot

Вариант 1 — отдельный чат с HermesBot:

```txt
Пользователь открывает HermesBot и пишет:
Привет, помоги составить план дня.
```

Вариант 2 — упоминание в группе:

```txt
@HermesBot подскажи идею для рекламы МДФ-фасадов.
```

Вариант 3 — команды:

```txt
/start
/help
/ask текст
/status
/reset
```

## Команды HermesBot

| Команда | Описание |
|---|---|
| `/start` | Приветствие и инструкция |
| `/help` | Список команд |
| `/ask` | Задать вопрос Hermes |
| `/status` | Проверить статус Hermes |
| `/reset` | Очистить контекст диалога |
| `/model` | Показать текущую модель |

## Контекст диалога

Для каждого пользователя или чата хранить небольшой контекст:

```txt
chat_id
last_messages
created_at
updated_at
```

Важно:

- не хранить слишком длинную историю;
- ограничить количество сообщений в контексте;
- очищать контекст по `/reset`;
- не отправлять чувствительные данные без согласия пользователя.

## Лимиты

Чтобы не перегружать Hermes:

- максимум 10 сообщений в минуту на пользователя;
- максимум 50 сообщений в час;
- максимум 1000 символов в одном запросе для MVP;
- таймаут ответа 60–120 секунд;
- если Hermes недоступен — вернуть понятное сообщение.

## Обработка ошибок

Если Hermes недоступен:

```txt
HermesBot: Сейчас Hermes недоступен. Попробуйте позже.
```

Если запрос слишком длинный:

```txt
HermesBot: Запрос слишком длинный. Сократите его до 1000 символов.
```

Если пользователь отправил команду:

```txt
HermesBot: Команда не найдена. Введите /help.
```

## Безопасность

Backend должен:

- проверять токен пользователя;
- проверять права доступа к чату;
- не передавать frontend-у пользователя напрямую в Hermes без проверки;
- логировать ошибки без токенов;
- ограничивать частоту запросов;
- хранить Hermes-ключ только в `.env`.

## Рекомендуемый первый вариант

Для первой рабочей версии:

1. HermesBot живёт в mock-режиме.
2. Frontend показывает команды и ответы.
3. Backend ещё не подключает настоящий Hermes.
4. После backend добавляется endpoint `/api/hermes/ask`.
5. Затем подключается Hermes gateway или API server.

## Что нужно для настоящего подключения

Перед настоящей интеграцией нужно решить:

- Hermes будет подключаться через gateway;
- Hermes будет подключаться через API server;
- Hermes будет подключаться через CLI;
- какой профиль Hermes использовать;
- какая модель должна быть у HermesBot;
- какие лимиты поставить.
