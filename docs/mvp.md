# MVP Hermes Messenger

## Что входит в первую рабочую версию

MVP должен показать интерфейс и логику мессенджера без полноценного backend.

## Экраны MVP

### 1. Список чатов

Функции:

- личные чаты;
- группы;
- каналы;
- боты;
- HermesBot;
- поиск по чатам;
- индикатор непрочитанных.

### 2. Окно чата

Функции:

- история сообщений;
- поле ввода;
- отправка сообщения;
- статус отправки;
- статус “печатает…”;
- ответы HermesBot;
- имитация WebSocket.

### 3. Группы

Функции:

- список участников;
- название группы;
- аватар;
- системные сообщения;
- имитация добавления участника.

### 4. Каналы

Функции:

- посты канала;
- счётчик просмотров;
- имитация подписки;
- администраторский режим.

### 5. Боты

Функции:

- HermesBot;
- команды;
- подсказки;
- примеры запросов;
- имитация ответа.

## Mock-режим

До backend можно хранить данные в:

```txt
localStorage
```

Что хранится:

- пользователи;
- чаты;
- сообщения;
- настройки;
- выбранный активный чат.

## HermesBot в MVP

В mock-MVP HermesBot не обращается к настоящему Hermes.

Он отвечает по правилам:

- `/start` — приветствие;
- `/help` — список команд;
- `/status` — статус системы;
- обычный текст — имитация ответа AI.

Пример:

```txt
Пользователь: Привет, Hermes
HermesBot: Привет! Я HermesBot. Сейчас работаю в mock-режиме.
```

## MVP без backend

Плюсы:

- быстро;
- можно показать интерфейс;
- можно открыть в браузере;
- можно опубликовать на GitHub Pages;
- можно завернуть в APK.

Минусы:

- нет реальной регистрации;
- нет настоящего realtime между разными устройствами;
- нет серверной базы;
- Hermes отвечает имитацией.

## После MVP

Когда интерфейс готов, добавляем backend:

- авторизация;
- SQLite;
- REST API;
- WebSocket;
- настоящая история сообщений;
- группы;
- каналы;
- HermesBot через backend-прокси.

## Критерии готовности MVP

MVP считается готовым, если можно:

- открыть страницу в браузере;
- выбрать чат;
- отправить сообщение;
- увидеть ответ в истории;
- переключаться между личным чатом, группой и каналом;
- открыть HermesBot;
- написать команду;
- получить ответ;
- обновить страницу и сохранить историю через localStorage.
